# LasyTeacher (LazyTeacher) — Анализ проекта

## Обзор проекта
**LasyTeacher** — это специализированная система управления образовательным контентом (CMS/LMS), разработанная на **Django 5.2.3**. Основная цель проекта — автоматизация работы учителя по ведению базы учебных задач, формированию контрольных и самостоятельных работ, а также генерации раздаточных материалов в различных форматах (HTML, PDF, LaTeX).

---

## Технологический стек
- **Backend:** Python 3.10+, Django 5.2.3
- **Frontend:** Django Templates, Vanilla CSS (планируется переход на BEM), JavaScript
- **Формы и UI:** `django-crispy-forms`, `crispy-bootstrap5`, `django-widget-tweaks`
- **Работа с изображениями:** `Pillow`
- **Генерация данных:** `Faker` (для тестирования)
- **Форматы вывода:** HTML, LaTeX, PDF

---

## Архитектура и основные модули

### 1. Ядро и контент
*   **`tasks` (Задания):** Центральный модуль системы. Поддерживает различные типы задач (качественные, расчетные, тесты), уровни сложности, кодификаторы и метаданные. Особенность — поддержка нескольких изображений с гибким позиционированием (`right_40`, `bottom_100` и т.д.).
*   **`curriculum` (Учебная программа):** Иерархическая структура: Предмет → Раздел → Тема → Подтема. Позволяет классифицировать задачи для удобного поиска и фильтрации.
*   **`task_groups` (Группы задач):** Механизм создания групп аналогов, что критично для генерации разных вариантов одной и той же работы.

### 2. Управление учебным процессом
*   **`works` (Работы и варианты):** Реализует логику "Шаблон работы → Генерация вариантов". Включает сложную систему распределения баллов (нормировка весов) и механизм случайного выбора задач из групп аналогов.
*   **`students` (Ученики):** Учет учеников и классов/групп.
*   **`events` (События):** Вероятно, модуль для проведения работ и фиксации результатов.

### 3. Генераторы документов
Проект обладает мощной подсистемой экспорта:
*   **`core_logic` + `infrastructure/services`:** Секционный document engine для HTML, LaTeX и PDF.
*   **`document_generator`:** Хранение шаблонов документов и compatibility-утилиты старого пути.

---

## Ключевые особенности
1.  **Гибкая генерация вариантов:** Учитель создает "спецификацию" работы, а система автоматически собирает нужное количество уникальных вариантов.
2.  **Многоформатность:** Поддержка LaTeX позволяет получать материалы высокого качества с математическими формулами.
3.  **Метаданные задач:** Глубокая классификация по уровням познания (Блум), источникам и элементам содержания (ФИПИ/ЕГЭ).
4.  **Умное позиционирование изображений:** Система учитывает, как изображения должны обтекаться текстом в печатных формах.

2.  **Интеграция MathJax/KaTeX:** Для качественного отображения формул в веб-интерфейсе.
3.  **API:** Возможно, стоит рассмотреть создание REST API (на базе DRF) для отделения фронтенда и мобильного приложения в будущем.
4.  **Тестирование:** Расширение покрытия тестами модулей генерации, так как они являются критическим функционалом.
